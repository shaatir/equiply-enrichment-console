Rules Overview

  • Sanitization: Inputs are cleaned and trimmed to eliminate layout spacing discrepancies.
  • Prefix Isolation: If empty, the serial is split on delimiters (-, _, or space) to extract the first token.
  • Lineage Tagging: Establishes clear metadata tracing.

Proprietary Brand Decoders

Edan Instruments M[YY]

Matches -M\d{2}. Extracts 2 digits as the 21st-century year (e.g. M19 $\rightarrow$ 2019).

Zoll Medical Alpha-Date

Matches ^[A-Z]{1,2}(\d{2})([A-L]). Translates the letter to a month (e.g. B $\rightarrow$ Feb, 14 $\rightarrow$ 2014).

Hill-Rom Model Fallback

If not decoded from the serial, evaluates models (e.g. Century $\rightarrow$ 1999-01-01, P3200 $\rightarrow$ 2016-01-01).

American Diagnostic C?[YY]

Matches ^C?(\d{2}) at start. Converts double digits to 4-digit years.

Standard Framework Integration

  • Standardized Nomenclature: Categorization names represent clean medical terminology (e.g. Aneroid Sphygmomanometer).
  • Precedence Filters: Regular expression keywords contain word boundaries (\b) to avoid overlap bugs (e.g., preventing a Pulse Oximeter model RAD8 from matching X-Ray's rad pattern).
  • Fallthrough Mode: If rules yield no matches, the row is marked as Unknown Device.

Audit Check & Confidence Cutoff

  • Point Deductions: Low confidence scores are caused by fallback index years (-35%), unknown categories (-30%), or brand default dates (-20%).
  • The Cutoff: Any record with a score of 80% or lower is automatically flagged for review.
  • Automation: Staged rows are submitted silently to the local endpoint in the background without halting UI interactions.

Queueing & Batching

  • Concurrent Queue: Front-end requests are gathered in a local Node queue (`createBatchQueue`).
  • JSON Schema Enforcement: The server batches records (size: `10`) and queries OpenAI using strict JSON schemas to prevent hallucinations.
  • Lineage Tag: Once resolved, the origin is changed to `'ai'`, and confidence scores are re-evaluated.
